CAMBIOS FRONTEND API - CodeFlow
Fecha: 2026-05-19

Objetivo:
Conectar el frontend React con la API Laravel que corre en http://127.0.0.1:8000/api.

Archivos creados o modificados:

1. src/lib/api.ts
- Centraliza las llamadas HTTP al backend.
- Usa VITE_API_URL si existe; si no, usa http://127.0.0.1:8000/api.
- Guarda el token en localStorage con la clave codeflow_token.
- Envia Authorization: Bearer TOKEN en rutas privadas.
- Utilidad: evita repetir fetch en cada pantalla.

2. src/context/AuthContext.tsx
- Reemplaza el login y registro simulados por llamadas reales a Laravel.
- Al cargar la app consulta /api/me si existe un token.
- Cierra sesion llamando /api/logout y limpiando localStorage.
- Actualiza perfil con PUT /api/me.
- Completa lecciones y retos usando los endpoints del backend.
- Utilidad: la sesion y el progreso ya pueden guardarse en base de datos.

3. src/routes/_app/leccion.$id.tsx
- Ahora espera a que el backend guarde la leccion antes de mostrar exito.
- Muestra error si no se pudo guardar.

4. src/routes/_app/retos.tsx
- Ahora espera a que el backend guarde el reto antes de mostrar exito.
- Muestra error si no se pudo guardar.

5. src/routes/_app/perfil.tsx
- Ahora guarda cambios de perfil contra el backend.
- Muestra error si no se pudo actualizar.

6. .env.example
- Define VITE_API_URL=http://127.0.0.1:8000/api como ejemplo de configuracion.

Comandos esperados cuando tengas Node/Bun disponible:

Instalar dependencias:
npm install

Levantar frontend:
npm run dev

O si usas Bun:
bun install
bun run dev

Nota:
En esta terminal no estan disponibles npm, node, pnpm ni bun, por eso no se pudo compilar o levantar el frontend desde aqui.
